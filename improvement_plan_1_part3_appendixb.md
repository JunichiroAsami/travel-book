# 改善案①：第3部と付録Bの役割分担と差別化案

**作成日時**：2025年11月27日  
**プロジェクト名**：ベトナム・タイ周遊記「AIと旅する、新しい冒険の形」  
**著者**：浅見 純一郎（あさみ じゅんいちろう）

---

## 1. 問題の整理

### 1.1. 3つのAIの指摘（全AI一致）

| AI | 指摘内容 |
|:---|:---|
| ChatGPT | 第4章の具体例 → 第7章の実践プロンプト → 付録B のテンプレート が、かなり重複しており、「ここはどこまで読めば十分なのか」が分かりづらい。 |
| Gemini | 第6〜9章（実践プロンプト集）と、付録B（プロンプトテンプレート集）は内容が密接に関連しており、読者によっては「同じことが繰り返されている」と感じる可能性があります。 |
| Claude | 第6章〜第9章は「プロンプト集」として構成されているが、付録Bも「プロンプトテンプレート集」である。読者にとって「どちらを使えばいいのか」が不明確になる可能性がある。 |

### 1.2. 問題の本質

1. **役割分担が不明確**：第3部と付録Bの違いが読者に伝わりにくい
2. **重複感**：同じプロンプトが第3部と付録Bの両方に登場する
3. **使い分けが不明**：「どちらを使えばいいのか」が分からない

---

## 2. 改善案の全体像

### 2.1. 役割分担の明確化

| 項目 | 第3部（第6章～第9章） | 付録B |
|:---|:---|:---|
| **目的** | ストーリーの中での使い方を学ぶ | 辞書的なリファレンスとして使う |
| **内容** | 実際のログを引用し、具体例を交えた解説 | テンプレート中心、網羅的 |
| **対象読者** | 初めてAIを使う読者、ストーリーを楽しみたい読者 | すぐに使いたい読者、特定のプロンプトを探している読者 |
| **読み方** | 順番に読む | 必要なときに参照する |
| **構成** | ストーリー性重視、代表例のみ | カテゴリー別、網羅的 |

### 2.2. デザインでの差別化

| 項目 | 第3部（第6章～第9章） | 付録B |
|:---|:---|:---|
| **フォント** | 通常のフォント | 等幅フォント（Courier New、Consolas） |
| **背景色** | 白 | 薄いグレー（#F5F5F5） |
| **枠線** | なし | あり（太さ1pt、色#CCCCCC） |
| **レイアウト** | 段落中心 | テンプレート形式（箇条書き中心） |
| **アイコン** | なし | 📝（プロンプト）、✅（チェックリスト） |

### 2.3. 相互参照の追加

**第3部**：
- 各プロンプトに「詳細は付録B-XX参照」と記載

**付録B**：
- 各プロンプトに「使用例は第X章参照」と記載

## 3. 具体的な改善内容

### 3.1. 第3部（第6章～第9章）の改善

#### 改善前（現状）

```markdown
### 型1：旅行ガイドブック作成の型

#### 導入部
旅行前に最も重要なのは、旅行先の情報を収集することです。市販のガイドブックは情報が古かったり、自分の興味に合わなかったりします。そんなとき、AIに自分専用のガイドブックを作成してもらうことができます。

#### 実例1：ホーチミンガイド作成（LOG005）
2025年7月28日、著者はホーチミン旅行ガイドブックを日本人向けに約4万字で図解入りPDFとして作成することを依頼しました。

**著者の質問**:
> ホーチミンの旅行ガイドブックを日本人向けに作成してください。
```

#### 改善後

```markdown
### 型1：旅行ガイドブック作成の型

#### 導入部
旅行前に最も重要なのは、旅行先の情報を収集することです。市販のガイドブックは情報が古かったり、自分の興味に合わなかったりします。そんなとき、AIに自分専用のガイドブックを作成してもらうことができます。

#### 実例1：ホーチミンガイド作成（LOG005）
2025年7月28日、著者はホーチミン旅行ガイドブックを日本人向けに約4万字で図解入りPDFとして作成することを依頼しました。

> ホーチミンの旅行ガイドブックを日本人向けに作成してください。
```

### 3.2. 付録Bの改善

#### 改善前（現状）

```markdown
### 1.1. 旅行ガイドブック作成

**プロンプトテンプレート**：
> [旅行先]の旅行ガイドブックを[対象読者]向けに作成してください。

**使用例**：
> ホーチミンの旅行ガイドブックを日本人向けに作成してください。
```

#### 改善後

```markdown
### 1.1. 旅行ガイドブック作成

📝 **プロンプトテンプレート**：
┌─────────────────────────────────────────────┐
│ [旅行先]の旅行ガイドブックを[対象読者]向けに │
│ 作成してください。                           │
└─────────────────────────────────────────────┘

✅ **使用例**：
> ホーチミンの旅行ガイドブックを日本人向けに作成してください。

📖 **使用例の詳細は第6章「型1：旅行ガイドブック作成の型」参照**

💡 **カスタマイズのヒント**：
- [旅行先]：具体的な都市名や地域名を入力
- [対象読者]：日本人、家族連れ、一人旅、カップルなど
- 追加情報：予算、日数、興味のあるジャンル（グルメ、歴史、自然など）
```

## 4. 実装方法

### 4.1. 第2章「本書を120%楽しむための『読み方』ガイド」に追記

#### 追記内容

```markdown
### 2.3. 第3部と付録Bの使い分け

本書には、プロンプトの「型」を紹介する箇所が2つあります。

#### 第3部（第6章～第9章）：ストーリーの中で学ぶ

第3部では、著者の実際の旅行体験を通じて、プロンプトの使い方を学びます。

**こんな読者におすすめ**：
- 初めてAIを使う方
- ストーリーを楽しみながら学びたい方
- プロンプトの背景や意図を理解したい方

**読み方**：
- 第6章から順番に読む
- 実際のログ（LOG005、LOG014など）を参照しながら読む

#### 付録B：辞書的なリファレンス

付録Bでは、プロンプトのテンプレートを網羅的に紹介します。

**こんな読者におすすめ**：
- すぐに使いたい方
- 特定のプロンプトを探している方
- カスタマイズのヒントが欲しい方

**読み方**：
- 必要なときに参照する
- カテゴリー別に探す（旅行計画編、グルメ編、トラブル対応編など）

#### 相互参照の活用

第3部と付録Bは相互に参照し合っています。

- **第3部から付録Bへ**：「詳細は付録B-XX参照」
- **付録Bから第3部へ**：「使用例は第X章参照」

この相互参照を活用することで、ストーリーとテンプレートの両方を効果的に学べます。
```

### 4.2. 付録Bの冒頭に「本付録の使い方」を追加

#### 追記内容

```markdown
## 本付録の使い方

### 付録Bの目的

付録Bは、旅行中に「今すぐ使いたい」と思ったときに、すぐに参照できる辞書的なリファレンスです。

### 第3部との違い

| 項目 | 第3部（第6章～第9章） | 付録B |
|:---|:---|:---|
| **目的** | ストーリーの中での使い方を学ぶ | 辞書的なリファレンスとして使う |
| **内容** | 実際のログを引用し、具体例を交えた解説 | テンプレート中心、網羅的 |
| **対象読者** | 初めてAIを使う読者、ストーリーを楽しみたい読者 | すぐに使いたい読者、特定のプロンプトを探している読者 |
| **読み方** | 順番に読む | 必要なときに参照する |

### 付録Bの構成

付録Bは、以下の6つのカテゴリーに分かれています。

1. **旅行計画編**：旅行前の計画に使えるプロンプト
2. **グルメ・レストラン編**：グルメやレストランに関するプロンプト
3. **観光・アクティビティ編**：観光やアクティビティに関するプロンプト
4. **移動・交通編**：移動や交通に関するプロンプト
5. **トラブル対応編**：トラブル時に使えるプロンプト
6. **帰国後・振り返り編**：帰国後の整理に使えるプロンプト

### 付録Bの使い方

1. **カテゴリーから探す**：目次から該当するカテゴリーを選ぶ
2. **プロンプトテンプレートをコピー**：📝マークのプロンプトをコピーする
3. **カスタマイズ**：[旅行先]や[対象読者]を自分の状況に合わせて変更する
4. **AIに入力**：カスタマイズしたプロンプトをAIに入力する
5. **使用例を参照**：第3部の使用例を参照して、より深く理解する

### デザインの見方

付録Bでは、以下のアイコンとデザインを使用しています。

- 📝：プロンプトテンプレート（枠線で囲まれている）
- ✅：使用例
- 📖：第3部への参照
- 💡：カスタマイズのヒント
```

## 7. 実装上の注意点

### 7.1. デザインの一貫性

- 付録Bのデザイン（枠線、背景色、フォント）は、すべてのプロンプトで統一する
- アイコン（📝、✅、📖、💡）は、すべてのプロンプトで統一する

### 7.2. 相互参照の正確性

- 第3部から付録Bへの参照は、正確なセクション番号を記載する
- 付録Bから第3部への参照は、正確な章番号を記載する

### 7.3. 読者への配慮

- 第2章と付録Bの冒頭の説明は、初めて読む読者にも分かりやすく記載する
- 専門用語は避け、平易な言葉で説明する

### 8.1. 改善案の要点

1. **役割分担の明確化**：第3部はストーリー、付録Bはリファレンス
2. **デザインでの差別化**：枠線、背景色、フォント、アイコン
3. **相互参照の追加**：第3部と付録Bを行き来できる

### 8.2. 実装スケジュール

- **合計所要時間**：6時間

- 読者の混乱を解消
- 使いやすさの向上
- 満足度の向上
