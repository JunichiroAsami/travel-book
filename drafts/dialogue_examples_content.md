# 第3章 対話型実例（執筆版）

## 第1節：対話型実例「白紙の旅程を、AIと共に設計する」

### 導入

航空券だけは予約した。しかし、それ以外は何も決まっていない。ホーチミンとバンコク、2つの大都市を巡る7日間。観光と仕事を両立させたい。できればホイアンにも立ち寄りたい。そして、現地のビジネスパートナーとの会食も予定されている。

この混沌とした状態から、どうやって一本の旅程を描き出せばいいのか。私はAIに最初の問いを投げかけました。

### 対話の再現

**私**: 「行きと帰りの飛行機を決めてますが、他決めてません。15日の夜に谷原さんと食事する以外決めてません。プランをいくつか考えて」

**AI**: 「ありがとうございます！8月12日に成田からホーチミンへ、8月19日にバンコクから成田に戻るフライトは把握しました。また、8月15日の夜に谷原さんとの食事も決まっているとのことですね。そこで、以下を教えていただけますか？

1. 滞在中に訪れたい都市や国はホーチミンとバンコク以外にもありますか？
2. 観光・仕事・リラックスなど、どのような目的で旅をされますか？
3. ホテルの予算帯や希望（エリア、タイプ、雰囲気など）はありますか？
4. 食事やアクティビティで特に興味のあるジャンル（例：ストリートフード、高級レストラン、マッサージ、歴史、アート）などありますか？
5. 移動手段（飛行機、列車、バスなど）やスケジュール感（移動は少なめが良い、日帰りOKなど）に希望はありますか？」

**私**: 「ホイアンが近ければ行きたい。基本的にホーチミンとバンコクで。観光と仕事で。ホテルは、ルネッサンス リバーサイド ホテル サイゴンを薦められた。マッサージと歴史かな。移動手段は、飛行機とグラブかな」

**AI**: 「ありがとうございます。ホーチミンとバンコクを中心に、観光と仕事を両立する旅程を組みつつ、ホイアンにも行ける可能性を含めた複数のプランを作成します。マッサージや歴史的スポットを含み、ホーチミンでは『ルネッサンス リバーサイド ホテル サイゴン』に基づいた滞在提案も行います。プランが整い次第、すぐにご提案いたします」

そして数分後、AIは3つの異なるテーマを持つ旅行プランを提示してくれました。それぞれのプランは、「文化体験重視」「リラックス重視」「ホイアン短期訪問」という明確な特徴を持ち、具体的な日程、アクティビティ、宿泊先の例、そして移動手段まで含まれていました。

### この対話から学べること

このやりとりで注目すべきは、**AIが質問を投げかけることで、私自身の旅の目的を明確化させた**点です。最初の私の依頼は非常に曖昧でしたが、AIの5つの質問に答えることで、自分が何を求めているのかが整理されました。

また、「ホイアンが近ければ行きたい」という漠然とした希望に対して、AIは「ホイアンにも行ける可能性を含めた複数のプラン」という形で対応しました。これは、AIが私の希望を無視せず、しかし現実的な制約（移動時間、滞在日数）も考慮した提案をしてくれたことを意味します。

最終的に、私はAIが提示した「文化体験重視プラン」をベースに、旅の全体像を固めていくことになります。AIが最初に提示したこの構造化された選択肢があったからこそ、その後の詳細な計画を効率的に進めることができました。

### AIとの対話で旅程を組み立てる際のポイント

1. **最初は曖昧でも構わない**: 「航空券以外何も決まっていない」という状態でも、AIに投げかけることで対話が始まる
2. **AIの質問に答えることで、自分の希望が明確になる**: 質問への回答プロセスが、思考の整理に役立つ
3. **複数の選択肢を求める**: 「プランをいくつか考えて」と依頼することで、比較検討が可能になる
4. **制約条件を伝える**: 「15日の夜に会食」「移動手段は飛行機とGrab」など、制約を明示することで、より現実的なプランが得られる
5. **漠然とした希望も伝える**: 「ホイアンが近ければ行きたい」のような曖昧な希望も、AIが適切に処理してくれる

### AIにホテル選びを依頼する際のポイント

1. **条件を具体的に列挙する**: 「評判がいい」「1泊1万円台」「朝食バイキング」など、優先順位の高い条件を明確に伝える
2. **比較表での提示を求める**: 複数の候補を表形式で比較することで、選択が容易になる
3. **追加検索を躊躇しない**: 「もっと他探して」という追加依頼は、選択肢を広げる有効な手段
4. **特定の候補について深掘りする**: 気になるホテル名を挙げることで、詳細な分析が得られる
5. **実際の宿泊者の声を求める**: レビューや評価は、意思決定の重要な材料となる

## 第3節：対話型実例「観光ルートを最適化し、段階的に詳細化する」

### 導入

観光スポットのリストは決まった。しかし、それらをどの順番で、どのように巡れば効率的なのか。移動時間、各スポットの開館時間、そして食事のタイミング。これらを考慮した最適なルートを組み立てるのは、地理に不案内な旅行者にとって大きな負担です。

私はホーチミンで訪れたい5つのスポットをリストアップし、AIに効率的な巡り方を尋ねました。そして、対話を通じて段階的に情報を深掘りしていくことで、単なるルート提案以上の価値を得ることができました。

### 対話の再現

**私**: 「これから以下を巡ろうかな。タンディン教会、ベトナム戦争証跡博物館、Independence Palace、Saigon Central Post Office、ニャハンゴン」

**AI**: 「いいですね、全て徒歩または短距離移動で回れるスポットです。以下の順路なら効率的に回れます。

**午前**
1. タンディン教会（ピンク教会）：朝の光で写真映えします。滞在目安：30分
2. ベトナム戦争証跡博物館：徒歩10分ほど。内容が重いので時間に余裕を持って。滞在目安：1〜1.5時間

**昼前後**
3. 独立宮（Independence Palace）：博物館から徒歩10分。ベトナム戦争期の歴史的建物。滞在目安：1時間

**午後**

AIは各段階で、私の質問に対して適切な粒度の情報を提供してくれました。特に、ATMに関する質問では、単に「ある」「ない」ではなく、「Wiseカード対応」「手数料無料のVPBank」「実際の体験談」「具体的なアドバイス」まで、実用的な詳細情報を提供してくれました。

### AIに観光ルート作成を依頼する際のポイント

1. **行きたい場所をリストアップする**: 具体的なスポット名を列挙することで、AIが地理的な位置関係を把握できる
2. **効率的な順路を求める**: 「どの順番で回るのが効率的か」と明示的に尋ねる
3. **地図リンクの作成を依頼する**: Googleマップリンクがあれば、現地での移動が格段に楽になる
4. **注意点を確認する**: 「注意点はあるか」と尋ねることで、現地特有の情報が得られる
5. **具体的な疑問を投げかける**: 「ATMはどこにあるか」など、実用的な疑問を段階的に深掘りする
