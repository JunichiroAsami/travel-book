# ドキュメントフォーマットガイドライン

## 1. 詳細設計書 (`detailed_design_template.md`)

### 1.1. 構成

1. **章タイトル**
2. **コンセプト**
3. **ターゲット読者**
4. **章の構成**
5. **各節の詳細**
6. **文字数目標**

### 1.2. 各セクションの記載事項

- **章タイトル**: 章のテーマを明確に表現する
- **コンセプト**: 章の目的と読者に伝えたいメッセージを記述
- **ターゲット読者**: 章が誰に向けたものかを明確にする
- **章の構成**: 章全体の流れを箇条書きで示す
- **各節の詳細**: 各節のテーマ、実例、学び、コツを具体的に記述
- **文字数目標**: 章全体の文字数目標を設定

## 2. 素材ファイル (`materials/chapters/chapterX_materials.md`)

### 2.1. 構成

1. **エピソード概要**
2. **状況**
3. **AIとの協業**
4. **結果**
5. **ログ参照箇所**

### 2.2. 各セクションの記載事項

- **エピソード概要**: エピソードの要点を簡潔にまとめる
- **状況**: エピソードが発生した背景や状況を説明
- **AIとの協業**: AIとどのように協力して問題を解決したかを記述
- **結果**: エピソードの結果、得られた成果を記述
- **ログ参照箇所**: 参照したログファイル名と該当箇所を明記

## 3. レビュー報告書 (`reviews/chapterX_review_roundY.md`)

### 3.1. 構成

1. **レビュー日**
2. **対象ドキュメント**
3. **総評**
4. **具体的な改善点**
5. **優れている点**

### 3.2. 各セクションの記載事項

- **レビュー日**: レビューを実施した日付を記載
- **対象ドキュメント**: レビュー対象のファイル名を記載
- **総評**: 全体的な印象と評価を記述
- **具体的な改善点**: 改善すべき点を具体的に指摘
- **優れている点**: 良かった点を3つ以上挙げる
