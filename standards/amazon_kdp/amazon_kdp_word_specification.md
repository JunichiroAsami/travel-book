# Amazon KDP用Word文書の仕様定義

**最終更新日**: 2025年12月4日

## 概要

このドキュメントでは、Amazon Kindle Direct Publishing (KDP)に提出するWord文書の仕様を定義します。本書「AI旅行術」のような技術書・実用書を想定しています。

## 1. 文書構造

### 1.1. 必須セクション

Amazon KDP用のWord文書には、以下のセクションが必要です：

1. **表紙・タイトルページ**（オプション、KDPで自動生成可能）
2. **目次**（自動生成、Wordの目次機能を使用）
3. **本文**
   - はじめに
   - 第1章〜第N章
   - おわりに
4. **付録**（オプション）
5. **著者紹介**（オプション）

### 1.2. 見出しレベル

本書では、以下の見出しレベルを使用します：

| レベル | 用途 | 例 |
|:---|:---|:---|
| **Heading 1** | 章タイトル、主要セクション | 「クイックスタートガイド」「はじめに」「第1章：...」「おわりに」「付録B：...」 |
| **Heading 2** | 章内の大セクション | 「2.1. 本書の独自性」「## 旅行計画編」 |
| **Heading 3** | 章内の中セクション | 「2.1.1. 実体験に基づくリアルな対話ログ」 |
| **Heading 4** | 章内の小セクション | 詳細な説明項目 |

### 1.3. Heading 1の一覧（本書の場合）

本書では、以下の17個のHeading 1を使用します：

1. クイックスタートガイド
2. はじめに
3. 第1章：なぜ今、あなたの旅に「AIという相棒」が必要なのか？
4. 第2章：本書を120%楽しむための「読み方」ガイド
5. 第3章：完璧な旅の設計士
6. 第4章：最強の現場司令塔
7. 第5章：冷静な危機管理コンサルタント
8. 第6章：旅行前の準備に使える「6つの型」
9. 第7章：旅行中に使える「実践プロンプト集」
10. 第8章：トラブル対応の「緊急プロンプト集」
11. 第9章：旅行後の振り返りに使える「分析プロンプト集」
12. 第10章：AI旅行の光と影
13. 第11章：AI時代の旅行者に求められる3つのスキル
14. 第12章：あなたの旅を変える第一歩
15. おわりに
16. 付録B：プロンプトテンプレート集
17. 付録C：AIツール活用ガイド

**注意**: 「第3部：実践編 - ベトナム・タイ周遊記」はHeading 2であり、Heading 1ではありません。

## 2. フォーマット要件

### 2.1. フォント

- **本文**: 明朝体（游明朝、MS明朝など）、10.5〜12pt
- **見出し**: ゴシック体（游ゴシック、MSゴシックなど）、太字
- **コード**: 等幅フォント（Consolas、Courier Newなど）

### 2.2. 行間・余白

- **行間**: 1.15〜1.5倍
- **段落間**: 0.5〜1行分
- **余白**: 上下左右 2.54cm（Wordのデフォルト）

### 2.3. ページ番号

- **位置**: ページ下部中央
- **開始番号**: 1（本文の最初のページから）
- **形式**: アラビア数字（1, 2, 3...）

### 2.4. 目次

- **生成方法**: Wordの自動目次機能を使用
- **深さ**: Heading 2まで（`--toc-depth=2`）
- **更新**: 提出前に必ず「フィールドの更新」を実行

## 3. マークダウンからの変換要件

### 3.1. 見出しの対応

マークダウンの見出しは、以下のようにWordの見出しスタイルに対応します：

| マークダウン | Wordスタイル |
|:---|:---|
| `# 見出し` | Heading 1 |
| `## 見出し` | Heading 2 |
| `### 見出し` | Heading 3 |
| `#### 見出し` | Heading 4 |

### 3.2. 特殊要素の扱い

| マークダウン要素 | Word変換後 |
|:---|:---|
| **太字** `**text**` | 太字 |
| *斜体* `*text*` | 斜体 |
| `コード` | 等幅フォント |
| コードブロック | 等幅フォント、背景色付き |
| 表 | Wordの表 |
| リスト | Wordの箇条書き |
| 水平線 `---` | **削除推奨**（Wordでは不要） |

### 3.3. 削除すべき要素

以下の要素は、Word変換前にマークダウンから削除することを推奨します：

1. **水平線** (`---`): Wordでは見出しの区切りが明確なため不要
2. **HTMLタグ**: Pandocが正しく処理できない場合がある
3. **過度な空行**: 段落間の空行は1行で十分

## 4. ファイル構成

### 4.1. 章ごとのマークダウンファイル

本書では、以下のファイル構成を使用します：

```
/home/ubuntu/ai_travel_book_project/finals/
├── quickstart.md              # クイックスタートガイド
├── preface.md                 # はじめに
├── chapter1_final.md          # 第1章
├── chapter2_final.md          # 第2章
├── chapter3_final.md          # 第3章
├── chapter4_final.md          # 第4章
├── chapter5_final.md          # 第5章
├── chapter6_final.md          # 第6章（第3部の導入を含む）
├── chapter7_final.md          # 第7章
├── chapter8_final.md          # 第8章
├── chapter9_final.md          # 第9章
├── chapter10_final.md         # 第10章
├── chapter11_final.md         # 第11章
├── chapter12_final.md         # 第12章
├── afterword.md               # おわりに
├── appendix_b_final.md        # 付録B
└── appendix_c_final.md        # 付録C
```

### 4.2. 統合マークダウンファイル

章ごとのファイルを結合した統合マークダウンファイル：

```bash
cat quickstart.md preface.md chapter1_final.md chapter2_final.md \
    chapter3_final.md chapter4_final.md chapter5_final.md \
    chapter6_final.md chapter7_final.md chapter8_final.md \
    chapter9_final.md chapter10_final.md chapter11_final.md \
    chapter12_final.md afterword.md appendix_b_final.md \
    appendix_c_final.md > complete_manuscript_v18.md
```

## 5. Amazon KDPの推奨事項

### 5.1. ファイル形式

- **推奨**: .docx（Word 2007以降）
- **非推奨**: .doc（Word 2003以前）

### 5.2. ファイルサイズ

- **推奨**: 50MB以下
- **本書の場合**: 約186KB（問題なし）

### 5.3. 画像

- **形式**: JPEG、PNG
- **解像度**: 300dpi以上推奨
- **サイズ**: 幅1000px以上推奨

### 5.4. ハイパーリンク

- **外部リンク**: 使用可能
- **内部リンク**: 目次から章へのリンクは自動生成

## 6. 検証チェックリスト

Word文書をAmazon KDPに提出する前に、以下を確認してください：

- [ ] Heading 1の数が正しい（本書の場合: 17個）
- [ ] 目次が正しく生成されている
- [ ] 目次を更新した（「フィールドの更新」を実行）
- [ ] ページ番号が正しく設定されている
- [ ] 誤字脱字がない
- [ ] 画像が正しく表示されている
- [ ] ハイパーリンクが正しく機能している
- [ ] ファイルサイズが50MB以下
- [ ] ファイル形式が.docx
