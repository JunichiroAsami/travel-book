## 第4章の実例1（コードブロック形式に変換）

### 変更後の表現（コードブロック形式）

```markdown
私は、AIに対して以下のように質問した。

```
【プロンプト】
バンコクのドンムアン空港から、ホテル「Oneday Hostel」まで、どうやって行けばいいですか？
```

AIは、以下のように回答した。

「ドンムアン空港からOnedayHostelまでのアクセス方法をいくつかご紹介します。

1. **タクシー**：所要時間約30分、料金200〜300バーツ
2. **Grab（配車アプリ）**：所要時間約30分、料金150〜250バーツ
3. **公共交通機関**：A1バス→BTSモーチット駅→BTSサパーンクワイ駅、所要時間約1時間、料金約50バーツ

初めての方には、Grabをおすすめします。」

このように、AIは複数の選択肢を提示し、それぞれの所要時間、料金、利便性を比較してくれた。私は、この情報をもとに、Grabを使ってホテルまで移動することにした。
```

## 第10章の実例（現在の引用形式）

### 現在の表現（引用形式）

```markdown
> **状況**: トゥクトゥクの運転手から、橋を渡るために追加料金を請求された。
> 
> **プロンプト**: 「タイのバンコクでトゥクトゥクに乗っています。橋を渡る際に、運転手から追加料金を請求されました。これは正当な料金ですか？タイ語で『その料金は不当です』と伝えるには、何と言えばいいですか？」
> 
> **AIの回答（一部抜粋）**:
> - **結論**: 橋の通行料の追加請求は一般的ではなく、基本的には不要な支払いです。
> - **交渉用のタイ語**:
>   - 『マイ・チャイ・カー・タン・ドゥアン（ไม่ใช่ค่าทางด่วน）』: 高速道路料金ではない、という意味。
>   - 『ペン・カー・ブリカーン・ルアム・ユー・แล้ว（เป็นค่าบริการรวมอยู่แล้ว）』: 料金に含まれているはずだ、という意味。
```

## 比較まとめ

### コードブロック形式の特徴
- ✅ プロンプトがコピー&ペーストしやすい
- ✅ 「プロンプト」という概念が視覚的に明確
- ✅ 技術書らしい雰囲気
- ✅ 読者が実際に使いやすい

### 引用形式の特徴
- ✅ 対話の流れがわかりやすい
- ✅ 読みやすい
- ❌ プロンプトのコピーがやや面倒

## 推奨
**コードブロック形式に統一**することで、本書の実践性が高まります。
