# 第3章本文作成の指示

## 1. 作成する内容
第3章「完璧な旅の設計士」の本文のみを作成してください。

**注意**: 付録Aは別途作成するため、本文のみを執筆してください。

## 2. 添付ファイル
以下のファイルを添付しています：

### 全体企画ファイル（2ファイル）
1. 書籍企画書（book_proposal.md）
2. 章構成マップ（chapter_structure_map.md）

### 第3章固有ファイル（3ファイル）
3. 詳細設計書（chapter3_detailed_design_v2.md）
4. 章別素材集（chapter3_materials_v2.md）- 各ログの要約
5. 出力例ファイル（chapter3_example.md）

## 3. 執筆の制約条件
- 想定文字数: 12,000～15,000文字
- 4節構成（詳細設計書参照）
- 実際のログの要約に基づく（想像や創作は一切なし）
- ハイブリッド形式（要約 + 重要部分の引用）

## 4. 各節の執筆ルール

### 導入部（300～400文字）
- 該当する段階の重要性を説明
- 従来の方法の限界を指摘
- AIがどのように役立つかを示す

### 実例（700～900文字 × 複数）
- 要約形式で状況を説明
- 著者の質問を引用ブロックで記載（素材集の要約から推測）
- AIの回答の重要部分を引用ブロックで記載（素材集の「解決策」を基に）
- 結果・考察を要約形式でまとめる
- トレーサビリティを記載（ログIDとファイル名）

### まとめ（200～300文字）
- 該当する段階におけるAIの役割をまとめる
- 読者に「AIを使ってみよう」と思わせる

## 5. 引用ブロックのスタイル

素材集には実際の対話全文は含まれていませんが、要約から推測して以下の形式で引用ブロックを作成してください：

```markdown
> **著者の質問**:
> 「（素材集の「問題・課題」から推測した質問内容）」

> **AIの回答**:
> 「（素材集の「解決策」を基にした回答内容）」
```

**重要**: 引用ブロックの内容は、素材集の要約に基づいて自然な対話形式に変換してください。想像や創作ではなく、要約から論理的に推測できる内容のみを記載してください。

## 6. トレーサビリティの記載
各実例の末尾に以下のように記載：
```
（LOG001：`2025-7-25 22-18-43-_______.md`より）
```

## 7. 企画書と章構成マップの参照方法

書籌企画書と章構成マップを参照し、以下の情報を把握してください：

- **ターゲット読者**: 誰に向けて書いているか
- **書籍の目的**: 読者に何を伝えたいか
- **トーン&マナー**: どのような文体で執筆するか
- **第3章の位置付け**: 書籍全体の中で第3章が果たす役割

これらの情報を踏まえ、書籍全体の文脈に沿った第3章を執筆してください。

---

## 8. 素材集の参照方法

章別素材集（chapter3_materials_v2.md）には、以下の情報が記載されています：

- **ログID**: 各ログの識別番号
- **日付**: 対話が行われた日付
- **ファイル名**: 実際のログファイル名
- **要約**: 対話の概要
- **問題・課題**: 著者が抱えていた問題
- **解決策**: AIが提案した解決策
- **キーワード**: 対話の主要なキーワード

これらの情報を基に、実際の対話を推測して本文を執筆してください。

## 8. 出力例の参照

出力例ファイル（chapter3_example.md）には、第1節の導入部と実例1の完全な例が記載されています。このスタイルを参考に、残りの実例を執筆してください。

## 9. 重要な注意点

- 素材集の要約のみを使用し、想像や創作は最小限に抑えること
- 引用ブロックの内容は、要約から論理的に推測できる範囲に留めること
- 読者が「自分も使ってみたい」と思える文体で執筆すること
- 専門用語を避け、平易な言葉で説明すること
- 各実例は独立しており、読者がどの実例から読んでも理解できるようにすること
