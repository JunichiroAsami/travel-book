# 第3章初稿作成の指示

## 1. 作成する内容
第3章「完璧な旅の設計士」の初稿と、付録A（第3章分）を作成してください。

## 2. 添付ファイル
以下のファイルを添付しています：

### 設計・ガイドファイル（4ファイル）
1. 詳細設計書（chapter3_detailed_design_v2.md）
2. 章別素材集（chapter3_materials_v2.md）- 各ログの要約
3. 出力例ファイル（chapter3_example.md）
4. この指示プロンプト（chapter3_prompt.md）

### 実際の対話ログファイル（17ファイル）
第3章で使用する17個のログファイルを添付しています。各ログファイルには、実際のChatGPTとの対話全文が記載されています。

素材集の各素材に記載されている「ファイル名」フィールドと、添付されているログファイル名を照合して使用してください。

## 3. 執筆の制約条件
- 想定文字数: 12,000～15,000文字
- 4節構成（詳細設計書参照）
- 実際のログに基づく（想像や創作は一切なし）
- ハイブリッド形式（要約 + 重要部分の引用）

## 4. 各節の執筆ルール
### 導入部（300～400文字）
- 該当する段階の重要性を説明
- 従来の方法の限界を指摘
- AIがどのように役立つかを示す

### 実例（700～900文字 × 複数）
- 要約形式で状況を説明
- 著者の質問を引用ブロックで記載
- AIの回答の重要部分を引用ブロックで記載
- 結果・考察を要約形式でまとめる
- トレーサビリティを記載（ログIDとファイル名）

### まとめ（200～300文字）
- 該当する段階におけるAIの役割をまとめる
- 読者に「AIを使ってみよう」と思わせる

## 5. 引用ブロックのスタイル
```markdown
> **著者の質問**:
> 「実際の質問内容」

> **AIの回答**:
> 「実際の回答内容（重要な部分のみ）」
```

## 6. トレーサビリティの記載
各実例の末尾に以下のように記載：
```
（LOG001：`2025-7-25 22-18-43-_______.md`より）
```

## 7. 付録Aの作成
### 7.1 作成内容
第3章で使用した全17ログの実際の対話全文をまとめた「付録A」を作成してください。

### 7.2 各ログの記載形式
```markdown
## LOG001: 旅行プラン提案

**対話日時**: 2025年7月25日 22:18  
**ファイル名**: `2025-7-25 22-18-43-_______.md`  
**使用AI**: ChatGPT 4o  
**本文での引用箇所**: 第3章 第1節 実例1（p.XX）

### 対話の背景
著者が8月中旬のホーチミンとバンコク旅行プランを検討していた際の対話です。観光と仕事の両立、ホイアンへの訪問も希望していました。

### 実際の対話

**著者**:
ホーチミンとバンコク中心の8月中旬旅行プランを提案してください。観光と仕事を両立させたい、ホイアンにも行きたいです。

**ChatGPT**:
承知しました。8月中旬のホーチミンとバンコクを中心とした旅行プランを提案します...

[実際の対話全文を掲載]

### このログから学べること
- AIに旅行プランを依頼する際の質問の仕方
- 複数の希望条件を伝える方法
- AIが提案するプランの構成
```

## 8. 対話ログの参照方法

### 8.1 ログファイルの照合
素材集（chapter3_materials_v2.md）には、各ログの要約が記載されています。実際の対話全文は、添付されている17個のログファイルを参照してください。

例：
- 素材集の「素材1」のファイル名が `2025-7-25 22-18-43-_______.md` の場合
- 添付ファイルの中から同じファイル名のファイルを開いて、対話全文を確認

### 8.2 対話の引用方法
ログファイルには、以下の形式で対話が記載されています：
```
## USER
（著者の質問）

## ASSISTANT
（ChatGPTの回答）
```

この形式を、本文では以下のように変換してください：
```markdown
> **著者の質問**:
> 「（著者の質問内容）」

> **AIの回答**:
> 「（ChatGPTの回答の重要部分）」
```

## 9. 重要な注意点
- 添付されている実際のログファイルを必ず参照すること
- 対話ログは省略せず、重要な部分を正確に引用すること
- 想像や創作は一切行わないこと
- 読者が「自分も使ってみたい」と思える文体で執筆すること
- 専門用語を避け、平易な言葉で説明すること
