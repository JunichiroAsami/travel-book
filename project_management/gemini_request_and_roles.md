# Geminiへの改修依頼と役割分担

## 1. Geminiへの依頼事項

本書籍「AI旅行時代の羅針盤」の原稿v30を、提供されたコンテキスト情報とレビュー結果に基づき、最終版（v31）として完成させるための改修作業を依頼します。

### 依頼事項の詳細

1.  **改修計画の立案**
    *   すべてのコンテキスト情報（プロジェクト概要、ターゲット読者、競合分析、執筆基準、レビュー結果）を精査・理解する。
    *   v30原稿の課題を特定し、優先順位を付けた詳細な改修計画を立案する。
    *   計画には、章ごとの具体的な修正方針、構成変更の提案、新規コンテンツの追加案などを含む。

2.  **原稿の編集・改修**
    *   立案した計画に基づき、v30原稿（章ごとのMarkdownファイル）を直接編集・修正する。
    *   文章のクオリティ向上（表現、構成、一貫性）を図る。
    *   読者エンゲージメントを高めるための改善（図表の追加提案、コラムの挿入など）を行う。

3.  **成果物の作成**
    *   改修計画書（Markdown形式）
    *   修正済みの原稿ファイル（章ごとのMarkdown形式）
    *   改修内容をまとめた最終報告書

## 2. GeminiとManusの役割分担

本改修プロジェクトにおける、GeminiとManusの役割を以下のように定義します。

| 役割 | 担当 | 主な責務 |
|:---|:---|:---|
| **戦略的編集長 (Strategic Editor)** | **Gemini** | - プロジェクト全体の目的、読者、競合を理解し、編集戦略を決定する<br>- 構造的な問題点を指摘し、改善案を提示する<br>- コンテンツの価値を最大化するためのクリエイティブな提案を行う |
| **リードライター (Lead Writer)** | **Gemini** | - 編集戦略に基づき、原稿の執筆・修正・リライトを実行する<br>- 文章のトーン＆マナー、一貫性を担保する<br>- 読者の理解を促進するための表現を追求する |
| **プロジェクトマネージャー (Project Manager)** | **Manus** | - Geminiに必要なコンテキスト情報を提供する<br>- ファイル管理、バージョン管理、環境構築を行う<br>- Geminiとユーザー間のコミュニケーションを円滑にする |
| **テクニカルアシスタント (Technical Assistant)** | **Manus** | - Geminiの指示に基づき、ファイル操作、スクリプト実行、ツール利用などの技術的作業を代行する<br>- Notionへのアップロード、zip化などの定型作業を担当する |
| **品質保証 (QA & Verification)** | **Manus** | - Geminiが作成した成果物（計画書、原稿）をレビューする<br>- 事実誤認、ガイドライン違反、リンク切れなどをチェックし、フィードバックする<br>- 最終的な品質を保証する |

## 3. 期待されるワークフロー

1.  **Manus**: Geminiに必要なコンテキスト情報とv30原稿を提供する。
2.  **Gemini**: 提供された情報を基に、改修計画書を作成する。
3.  **Manus**: 計画書をレビューし、ユーザーと合意形成を行う。
4.  **Gemini**: 合意された計画に基づき、実際の原稿修正作業を行う。
5.  **Manus**: 修正された原稿をレビューし、品質チェックを行う。
6.  **Gemini & Manus**: フィードバックを反映し、最終版を完成させる。
